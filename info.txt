# Обзор функционала приложения (для ТЗ и лендинга)

## 1. Общее описание
Веб-сервис для автоматизации закупок ресторанов. Позволяет загружать прайс-листы поставщиков, искать товары по единой базе, сравнивать цены (тендеры), формировать заказы и управлять доступами сотрудников.

## 2. Ролевая модель
*   **Администратор платформы (Admin)**: Полный доступ, управление всеми пользователями, возможность "имперсонации" (вход под любым пользователем).
*   **Администратор компании (Company Admin)**: Управление сущностями своей компании (рестораны, пользователи, поставщики).
*   **Пользователь (User)**: Ограниченный доступ к назначенным ресторанам. Работа с поиском, корзиной и тендерами.

## 3. Основные модули и возможности

### 3.1. Поиск и Каталог (Search)
*   **Единый поиск**: Мгновенный поиск товаров по прайс-листам всех подключенных поставщиков.
*   **Фильтрация и Сортировка**:
    *   Фильтр по конкретному поставщику.
    *   Сортировка: по релевантности, цене (возрастание/убывание).
    *   Лимит выдачи результатов.
*   **Карточка товара**: Отображение названия, поставщика, единицы измерения, цены.
*   **Быстрое добавление**: Возможность добавить товар в корзину/заказ прямо из поиска с указанием количества.

### 3.2. Корзина и Заказы (Cart)
*   **Сводная корзина**: Товары сгруппированы по поставщикам.
*   **Редактирование**: Изменение количества, удаление позиций.
*   **Экспорт**:
    *   Скачивание заказа в Excel (.xlsx).
    *   Скачивание в CSV.
    *   (Функционал отправки заказа по API/Email в разработке, сейчас реализован экспорт файлов).

### 3.3. Тендеры и Сравнение цен (Tenders)
Инструмент для проведения закупочных сессий и выбора оптимального поставщика.
*   **Создание проекта**: Тендер создается под конкретный ресторан.
*   **Загрузка списка**:
    *   Вставка списка продуктов текстом (копипаст из мессенджеров/заметок).
*   **Подбор поставщиков**: Выбор поставщиков для участия в сравнении.
*   **Умный матчинг (Matching)**:
    *   Система нечеткого поиска (Fuzzy Search) для сопоставления товаров из списка с позициями прайс-листов.
    *   Ручной контроль: возможность выбрать наиболее подходящий аналог из предложенных вариантов (с указанием цены и схожести).
*   **Аналитика**: Сравнение цен, подсчет итоговой стоимости корзины у разных поставщиков.
*   **Выбор победителя**: Отметка лучших позиций (зеленая корзина), лучших цен (золотая цена).
*   **Генерация заказов**: Автоматическое формирование заказов поставщикам на основе выбранных позиций.

### 3.4. Управление Компанией (Lists / Company Dashboard)
*   **Рестораны**: Создание и редактирование списка ресторанов сети.
*   **Пользователи**:
    *   Приглашение сотрудников по Email.
    *   Назначение ролей.
    *   Разграничение прав доступа к конкретным ресторанам.
    *   Сброс паролей.
*   **Поставщики и Прайс-листы**:
    *   Добавление поставщиков.
    *   Назначение поставщиков для конкретных ресторанов.
    *   **Загрузка прайсов**: Импорт Excel-файлов. Два режима: "Полная замена" (актуализация) и "Дополнение".
    *   **Интеграции**: Встроенная поддержка автоматического получения прайсов от крупных поставщиков (Sweetlife, GFC) через логин/пароль.

### 4. Технические особенности
*   **Платформа**: Веб-приложение (PWA ready), адаптировано под мобильные устройства.
*   **Безопасность**:
    *   Система аккаунтов.
    *   Интеграция с Telegram для привязки аккаунта и входа.
*   **Производительность**: Быстрый поиск благодаря использованию полнотекстового индекса (PostgreSQL TSVector + pg_trgm).

## 5. Стек технологий (для справки)
*   **Backend**: Python, Flask.
*   **Database**: PostgreSQL.
*   **Frontend**: HTML5, Vanilla JS, CSS3 (Modern UI, Glassmorphism design).
*   **Infrastructure**: Docker, Caddy.
